# SVERA Website — Agent Instructions

## What This Is
**SVERA — Svenska Evenemang & Racerbåtsarkivet** (svera.nu)
A static website for Sweden's collected information platform for powerboat racing.
Replaces the old Svenska Racerbåtförbundet site with an independent archive/information resource.

## Architecture
```
svera-website/
  *.html              Static pages
  assets/css/style.css Main stylesheet
  assets/js/main.js   Client-side interactions (nav, dropdowns, filters)
  config.json          Credentials (NEVER commit)
  .htaccess            Redirects (/api → api.html, /docs → /api)
  bot/
    svera_daemon.sh    Main daemon (runs scrapers, builders, email worker)
    update.sh          Manual full scrape + deploy script
    deploy.sh          SFTP deploy to one.com
    email_worker.py    Email-to-AI pipeline (Charlie Webber)
    scrape_tracker.py  Tracks scrape timestamps (avoids redundant work)
    scrapers/          Data scrapers
    builders/          Page generators
    data/              Cached JSON from scrapers (gitignored)
```

## News Card Format (index.html)
News cards go in `<main id="nyheter"><div class="news-list">`. Newest first.
```html
<article class="news-card" style="border-left:4px solid FÄRG;">
  <div class="news-body">
    <div class="news-meta">
      <span class="category" style="background:FÄRG;color:#fff;">Kategori</span>
      <span class="separator"></span>
      <span class="news-date">YYYY-MM-DD</span>
    </div>
    <h3>Rubrik</h3>
    <p>Text</p>
  </div>
</article>
```

### Category Badge Colors
| Category | Badge color | Border color |
|----------|-----------|-------------|
| Evenemang | `#c0392b` (red) | `#c0392b` |
| Internationellt | `#e67e22` (orange) | `#e67e22` |
| Ny funktion | `linear-gradient(135deg,#ffd700,#f0c800)` color:`#5a4700` | `#ffd700` |
| Tävling | `#2980b9` (blue) | `#2980b9` |
| Nyhet/Arkivet/Klasser | Default yellow (no inline style) | Default |

### Rules
- `news-date` = today's date (publish date), NOT event date
- Event dates go in the text body
- No `<ul>/<li>` in news cards — use inline text with `&middot;` separator
- No `href='#'` — skip `<a class='read-more'>` if no real URL
- New cards go FIRST in `div.news-list`

## Calendar (kalender.html)
- SVEMO events in `<tbody id="svemo-events">` — rebuilt by build_kalender.py
- UIM events in `<tbody id="uim-events">` — rebuilt by build_kalender.py
- Two filter groups: Branch (Alla/Offshore/Rundbana/Aquabike/UIM) + Country (Sverige/Utomlands)
- Swedish events: `data-country="sweden"`, foreign: `data-country="other"`
- Foreign invitation events defined in `FOREIGN_EVENTS` list in build_kalender.py
- Merged events: same date + organizer → single row with multiple branch badges

## Email Worker (bot/email_worker.py)
Charlie Webber — email-to-AI pipeline:
- LOW tasks (no names) → DeepSeek V3
- LOW tasks (names) or MEDIUM → Qwen ZDR
- HIGH → Claude Code CLI
- Content creation tasks (inlägg, posts) auto-escalated to MEDIUM
- Always deploys after changes: `bash bot/deploy.sh`

## Design Rules
- Colors: `--primary: #253686`, `--accent: #fde506`, dark nav, light content
- Font: Titillium Web
- Mobile-first responsive, pure HTML/CSS/JS (no frameworks)
- All content in Swedish (`lang="sv"`)
- No emojis unless user requests them

## Deployment
- Host: one.com (free plan), SFTP only (no SSH)
- Creds in config.json, deploy via `sshpass`
- Clean URLs: /api → api.html, /docs → /api (via .htaccess)

## Security
- NEVER expose config.json or API keys
- Email on kontakt.html is JS-assembled (bot-proof)
- Filter out PB (Patrol Boat) and R (Rescue) from race results
- Social scrapers use public OG metadata — no API keys needed
